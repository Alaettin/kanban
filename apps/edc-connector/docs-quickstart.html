Diese Anleitung fuehrt Sie Schritt fuer Schritt durch den kompletten Datenaustausch-Prozess: vom Erstellen eines Assets bis zum Abrufen der Daten. Alle Schritte koennen direkt in der EDC Connector App ausgefuehrt werden.
Provider
1
Docker Container starten
Landing-Menue

Klicken Sie auf "Both" um den Split-Screen zu oeffnen, dann auf den gruenen Start-Button in der Status-Bar.

Warten Sie, bis beide Status-Punkte gruen werden:

Provider aktiv    Consumer aktiv
Tipp
Der erste Start kann 1-2 Minuten dauern, da das Docker Image gebaut wird. Bei nachfolgenden Starts geht es deutlich schneller.
2
Asset erstellen
Provider → Assets Tab

Klicken Sie "Asset erstellen" und fuellen Sie die Felder aus:

Asset ID product-data-1
Name Produktdaten API
Content Type application/json
Base URL https://jsonplaceholder.typicode.com/todos/1

Klicken Sie "Erstellen" — das Asset erscheint in der Liste.

JSON
{
  "@context": {"@vocab": "https://w3id.org/edc/v0.0.1/ns/"},
  "@id": "product-data-1",
  "properties": {
    "name": "Produktdaten API",
    "contenttype": "application/json"
  },
  "dataAddress": {
    "type": "HttpData",
    "baseUrl": "https://jsonplaceholder.typicode.com/todos/1"
  }
}
Tipp
Die Base URL kann jede beliebige HTTP(S)-Datenquelle sein — z.B. eine REST API, eine statische JSON-Datei oder ein interner Microservice.
3
Policy erstellen
Provider → Policies Tab

Klicken Sie "Policy erstellen" und konfigurieren Sie:

Policy ID open-policy-1
Typ Open

Der Typ Open bedeutet: keine Einschraenkungen — jeder Consumer darf zugreifen. Klicken Sie "Erstellen".

Tipp
In einer produktiven Umgebung wuerden Sie hier Zugriffsregeln definieren, z.B. bestimmte Business-Partner-IDs oder Zeitbeschraenkungen.
4
Contract Definition erstellen
Provider → Contract Defs Tab

Klicken Sie "Contract Def erstellen" und fuellen Sie die Felder aus:

Contract Def ID contract-def-1
Access Policy ID open-policy-1
Contract Policy ID open-policy-1
Asset ID Filter (leer lassen fuer alle Assets)

Klicken Sie "Erstellen". Die Contract Definition verknuepft Asset(s) mit den Zugriffs- und Vertragsregeln.

Tipp
Ohne Asset ID Filter gilt die Contract Definition fuer alle vorhandenen Assets. Fuer gezielte Zuordnung tragen Sie eine spezifische Asset ID ein.
Consumer
5
Katalog abfragen
Consumer → Catalog Tab

Die Counter Party Address sollte bereits vorausgefuellt sein:

Counter Party Address http://provider:19194/protocol

Klicken Sie "Katalog abfragen". Das Asset product-data-1 erscheint mit der zugehoerigen Policy.

Tipp
Der Katalog zeigt nur Assets, fuer die eine passende Contract Definition existiert. Wenn nichts angezeigt wird, pruefen Sie die Schritte 2-4.
6
Vertrag verhandeln
Consumer → Negotiations Tab

Klicken Sie "Verhandeln" auf dem Katalog-Eintrag. Die Felder werden automatisch ausgefuellt.

Klicken Sie "Starten" und wechseln Sie zum Negotiations Tab. Der Status durchlaeuft folgende Phasen:

REQUESTED → AGREED → VERIFIED → FINALIZED
Tipp
Die Verhandlung laeuft automatisch ab. Aktualisieren Sie die Ansicht, falls der Status nicht sofort wechselt — der Prozess dauert in der Regel wenige Sekunden.
7
Transfer starten
Consumer → Transfers Tab

Bei einer FINALIZED Verhandlung: Klicken Sie "Transfer starten". Die Felder werden automatisch ausgefuellt.

Klicken Sie "Starten" und wechseln Sie zum Transfers Tab.

REQUESTED → STARTED

Klicken Sie "Daten abrufen" — Sie erhalten einen EDR-Endpunkt mit Authorization-Token, ueber den die Daten abgerufen werden koennen.

Tipp
Der EDR (Endpoint Data Reference) enthaelt die URL und den Authorization-Token. Dieser Token ist zeitlich begrenzt — starten Sie bei Bedarf einen neuen Transfer.
Geschafft!
Sie haben erfolgreich Daten ueber den Eclipse Dataspace Connector ausgetauscht! Der Provider hat ein Asset bereitgestellt, und der Consumer hat es ueber das Dataspace Protocol verhandelt und abgerufen.
